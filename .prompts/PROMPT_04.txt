Canonical fixture layout convention:
- `fixtures/<source_id>/<fixture_id>/bundle.json` (metadata + parsed records)
- `fixtures/<source_id>/<fixture_id>/raw/` (one or more checked-in raw artifacts, e.g. `listing.html`, `detail.html`, `response.json`)
- `manual/<source_id>/` may contain symlinks/copies/wrappers that point to the same bundle format, but the schema stays identical

Manual/fixture JSON minimal schema (per file):
- fixture_id: string
- source_id: string
- crawlability: "Gated" | "ManualOnly" | "PublicHtml" | "Api" | "Rss"
- captured_from_url: string
- fetched_at: ISO-8601 timestamp
- extractor_version: string
- raw_artifact:
  - content_type: string
  - path: relative path to checked-in raw artifact file (preferred, relative to bundle location) OR inline_text
  - sha256: string (if known; compute if missing during seed/import)
- parsed_records: list of parsed opportunity records (fixture-derived)
- evidence_coverage_percent: number
- notes: optional string

Each parsed record must include provenance-compatible evidence fields sufficient to build `Field<T>` + `EvidenceRef`.
